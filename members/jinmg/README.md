## 기능 요약

| 기능 | 설명 |
|---|---|
| **메타데이터 워터마크 탐지** | EXIF/PNG 메타데이터에서 Midjourney·DALL-E·SD 등 30개 이상 AI 도구 서명 탐지 |
| **코너 워터마크 탐지** | 이미지 4개 모서리에서 텍스트/로고 패턴 분석 |
| **ViT 앙상블** | 3개 HuggingFace 분류 모델의 가중 투표 |
| **FFT 격자 분석** | 주파수 도메인의 AI 특유 격자 패턴 탐지 |
| **Gram 질감 분석** | 특징 맵 상관관계로 AI 특유 질감 균질성 탐지 |
| **SRM/Patch 노이즈** | 조작 흔적 잔차 및 패치 분산 이상 탐지 |
| **얼굴 특화 분석** | 검출된 얼굴 영역에 ViT를 집중 적용 |
| **오디오 분석** | 스펙트럼 평탄도로 합성 음성 탐지 |
| **C2PA 디지털 지문** | Content Credentials 표준 메타데이터 우선 신뢰 |
| **Instagram 스트리밍** | yt-dlp로 직접 스트림 URL 추출 (파일 저장 없음) |
| **Gradio UI** | 브라우저 기반 UI, 미리보기·FFT 스펙트럼·상세 리포트 제공 |

## 탐지 파이프라인 상세

### 0단계: 메타데이터 워터마크 1차 필터

AI 생성 도구는 출력 파일에 소프트웨어 이름, 생성 프롬프트, 모델 버전 등을 메타데이터로 기록합니다. 이 단계에서 탐지되면 **나머지 분석 없이 즉시 AI로 확정**합니다.

#### 메타데이터 서명 탐지 (`check_metadata_watermark`)

```
탐지 대상 필드
├─ JPEG EXIF  (Software, ImageDescription, UserComment, Author 등)
└─ PNG/WebP tEXt 청크 (parameters, Comment, Description 등)
```

탐지 키워드 (30개 이상):

```
midjourney, dall-e, dall·e, stable diffusion, novelai, nijijourney,
adobe firefly, ideogram, runway ml, pika labs, kling, sora, imagen,
comfyui, automatic1111, invokeai, fooocus, diffusion,
ai generated, ai-generated, generated by ai, created by ai,
artificially generated, synthetic image, ...
```

결과: 발견 시 `100.0` 반환 → `compute_verdict`에서 즉시 확정

#### 코너 워터마크 탐지 (`check_corner_watermark`)

이미지의 4개 모서리(`min(H,W)/7` 크기)를 각각 분석합니다.

```
판단 조건 (하나의 코너):
  분산(var) < 800          → 배경이 단색에 가까움 (워터마크 배경 가능성)
  AND
  엣지 비율 1.5% ~ 25%    → 고대비 텍스트/로고가 존재함

2개 이상 코너가 조건 충족 → 워터마크 의심 점수 반환 (최대 70점)
```

### 1단계: 앙상블 ViT 추론

#### 사용 모델

| 모델 | 가중치 | 특징 |
|---|---|---|
| `Organika/sdxl-detector` | 0.40 | SDXL 계열 AI 이미지 특화 |
| `Nahrawy/AIorNot` | 0.35 | 범용 AI/실사 이진 분류 |
| `haywoodsloan/ai-image-detector-deploy` | 0.25 | 배포 특화 경량 모델 |

#### 라벨 분류 (`_label_is_ai`)

모델마다 출력 라벨 형식이 다르므로 키워드 기반으로 정규화합니다.

```python
# 부정 패턴을 먼저 확인 — "Not AI"가 'ai' 키워드를 포함해도 Real로 정확히 분류
_NEGATE_PATTERNS = ('not ai', 'not artificial', 'not fake', ...)
_AI_KEYWORDS     = ('ai', 'artificial', 'fake', 'generated', 'sdxl', ...)
_REAL_KEYWORDS   = ('real', 'human', 'natural', 'photo', ...)
```

> **핵심 버그 수정:** `"Not AI"` 라벨은 `'ai'` 문자열을 포함하므로 부정 패턴을 먼저 체크하지 않으면 AI로 잘못 분류됩니다. 이것이 이전 버전의 주요 인식률 저하 원인이었습니다.

#### 가중 앙상블 (`ensemble_vit_score`)

```
각 모델의 AI 확률 = AI 예측 점수 합 / (AI + Real 점수 합)
최종 점수 = Σ(모델별 AI 확률 × 가중치) / Σ(가중치) × 100
```

#### 얼굴 특화 분석

얼굴이 검출된 경우 얼굴 영역(패딩 30%)을 크롭하여 별도 ViT 추론을 수행합니다.

```
최종 ViT 점수 = 얼굴 영역 ViT × 0.7 + 전체 이미지 ViT × 0.3
```

AI 생성 인물 이미지에서 얼굴 부분에 아티팩트가 집중되는 특성을 활용합니다.

### 2단계: FFT 격자 분석

AI 이미지 생성 모델(특히 Diffusion 계열)은 주파수 도메인에 **규칙적인 격자 패턴**을 남깁니다. 이는 업샘플링/컨볼루션 연산의 주기적 특성에서 기인합니다.

#### `fft_grid_score` 처리 흐름

```
그레이스케일 이미지
    → 2D FFT (np.fft.fft2)
    → 주파수 중심 이동 (fftshift)
    → 로그 스케일 크기 스펙트럼 (20·log|F|)
    → 저주파 마스크 제거 (반경 60px 원형 마스크)
    → 이진화 (임계값 210)
    → 연결 요소 분석으로 피크 위치 추출
    → 대칭성·정렬도 계산 → grid 점수
    → peak_ratio (최대값/평균값) 계산
```

#### 판정 규칙

| 조건 | 결과 |
|---|---|
| `grid > 70` AND `peak_ratio > 2.8` | 격자 확정, 90+ 점 반환 |
| `grid < 35` AND `peak_ratio > 3.0` | 단일 피크 (최대 30점) |
| 그 외 | `max(grid, peak_ratio×4)` (최대 45점) |

### 3단계: 질감 및 노이즈 분석

#### Gram 행렬 질감 균질도 (`get_gram_std`)

```
특징 맵 F (shape: B×C×H×W)
    → 채널 차원으로 펼치기 (B×C×HW)
    → Gram = F × Fᵀ / (C×H×W)  (자기 상관 행렬)
    → std(Gram) 계산
```

- **낮은 std** → 채널 간 균질한 텍스처 → AI 의심
- 점수 계산: `(30 - gram_mean) × 2.5` (gram_mean < 30일 때만 양수)

#### SRM 잔차 (`srm_residual`)

카메라 센서 노이즈 패턴을 모델링한 Spatial Rich Model 커널로 잔차를 추출합니다.

```
커널 = [[-1, 2, -1], [2, -4, 2], [-1, 2, -1]] / 4
잔차 = std(filter2D(gray, kernel))
점수 = (잔차 - 12) × 5  (최대 95)
```

#### 패치 분산 이상 (`patch_anomaly`)

```
16×16 패치의 지역 분산 → 분산들의 표준편차 계산
높은 값 = 패치 간 질감 일관성 없음 = 콜라주/합성 의심
점수 = (std - 50) / 5  (최대 90)
```

### 4단계: 오디오 아티팩트 분석

딥페이크 영상에 합성 음성이 포함된 경우 탐지합니다.

```python
# 처음 10초 로드 → 스펙트럼 평탄도(Spectral Flatness) 계산
flatness = mean(librosa.feature.spectral_flatness(y))
score = (0.05 / (flatness + 1e-3)) × 10
```

- 합성 음성: 특정 주파수 대역이 지나치게 균일 → 낮은 flatness → 높은 score
- 자연 음성: 주파수 분포가 불규칙 → 높은 flatness → 낮은 score
- URL 스트림/이미지의 경우 librosa가 실패해도 `(0.0, "오디오 분석 불가")`로 graceful fallback

### 5단계: C2PA 디지털 지문

[Content Credentials (C2PA)](https://c2pa.org/) 표준은 카메라·편집 소프트웨어가 파일에 디지털 서명을 삽입하는 방식으로, 위조가 불가능합니다.

```python
reader = c2pa.Reader(source)
manifest = reader.json()
# "ai" 키워드 포함 → 100.0 (AI 확인)
# 매니페스트 존재하지만 AI 없음 → 0.0 (원본 확인)
# 매니페스트 없음 → None (정보 없음)
```

C2PA 결과가 있으면 최종 판정에서 **70% 가중치**로 시각 분석을 압도합니다.

### 최종 판정 로직

`compute_verdict` 함수는 위 모든 신호를 종합합니다.

#### 판정 트리

```
meta_wm_score ≥ 100
    → 즉시 99.0% 확정

face_detected = True
    p_vit ≥ 80  → 75% ViT + 15% max(Gram,FFT) + 10% 워터마크
    p_vit ≤ 30  → 65% ViT + 15% Gram + 10% FFT + 10% 워터마크  (실사 우호)
    30 < p_vit < 80
        grid_verified AND p_gram > 50  → max(p_vit, 70, wm)
        그 외                          → 55% ViT + 20% Gram + 15% FFT + 10% wm

face_detected = False
    grid_verified AND p_gram > 50  → max(p_fft, p_vit, 85, wm)
    p_vit < 35 AND not grid AND wm < 30  → 실사 우호 가중 평균
    그 외 → 동적 가중치 (ViT 40~50%, FFT 8~18%, Gram 12%, wm 10%)

c2pa_prob 존재  → 70% C2PA + 30% 위 계산값 (최우선)

최종값 → clamp(0, 99.9)
```

#### 워터마크 통합 신호

```python
wm_signal = max(meta_wm_score, p_corner_wm)
```

메타데이터 워터마크와 코너 워터마크 중 더 높은 값을 단일 신호로 통합합니다.

## Instagram URL 스트리밍 분석

파일을 저장하지 않고 직접 스트리밍 방식으로 분석합니다.

```
Instagram URL
    │
    ▼
get_stream_url()
    · yt-dlp extract_info(download=False)
    · formats 배열에서 MP4 우선 선택
    · 직접 CDN 스트림 URL 반환
    │
    ▼
cv2.VideoCapture(stream_url)
    · CAP_PROP_FRAME_COUNT == 0 → 스트리밍 모드 감지
    · 순차 읽기 (매 5프레임 샘플, 최대 10개)
    │
    ▼
첫 프레임 → _save_frame_preview() → 임시 JPG → Gradio 미리보기
```

**스트리밍 vs 파일 분석 차이점:**

| 항목 | 파일 | URL 스트림 |
|---|---|---|
| 프레임 접근 | 랜덤 seek (CAP_PROP_POS_FRAMES) | 순차 읽기 |
| 메타데이터 | EXIF/PNG 검사 가능 | 불가 |
| 오디오 분석 | librosa 직접 로드 | 실패 시 스킵 |
| 미리보기 | 원본 파일 반환 | 첫 프레임 JPG 임시 저장 |
| 임시 파일 | 없음 | 프레임 preview JPG 1개 |

## 알려진 한계

| 항목 | 내용 |
|---|---|
| **Instagram 비공개 게시물** | 로그인 없이 스트림 URL 추출 불가 |
| **Instagram CDN 만료** | 추출한 스트림 URL은 수 시간 후 만료됨 |
| **최신 AI 모델** | ViT 모델이 학습하지 않은 신규 생성 모델에 취약 |
| **고압축 영상** | JPEG/H.264 압축이 심하면 FFT·SRM 신호 희석 |
| **오디오 스트리밍** | librosa는 HTTP 스트림 직접 지원이 불안정하여 스킵될 수 있음 |
| **코너 워터마크** | OCR 없이 패턴 분석만 수행하므로 False Positive 가능성 존재 |
| **C2PA 보급률** | 아직 메타데이터가 삽입된 콘텐츠가 제한적 |
